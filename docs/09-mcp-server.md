# 09. MCP Server (gigflow-mcp)

経理担当者の Claude Desktop / VS Code / Cursor / Copilot Studio から接続される **MCP (Model Context Protocol) サーバ**。`packages/mcp-server/` に独立実装し、Container Apps にホストする。

> **設計思想**: gigflow を「経理向け AI から見たツール」として外部に公開する。書込はしない (read-only) ため、送金の二重実行リスクはゼロ。Multi-agent エコシステムの**最も重要な接続点**。

---

## 1. プロトコル・トランスポート

- **MCP 仕様**: 2025-06-18 版以降 (Streamable HTTP transport)
- **ベース URL**: `https://mcp-gigflow-{suffix}.{region}.azurecontainerapps.io/mcp`
- **トランスポート**: Streamable HTTP (SSE フォールバック有り)
- **認証**: Microsoft Entra ID (Bearer JWT)
- **テナント分離**: トークンの `tid` claim を `companyId` として使用

### 1.1 SDK
- `@modelcontextprotocol/sdk` (公式 TypeScript SDK)
- サーバ実装は Node.js 20 + Hono (HTTP layer)

## 3. Tools 一覧

すべての tool は **読み取り専用**。Cosmos の `companyId = ctx.tenantId` でテナント絞り込み必須。

### 3.1 `queryOrders`

**Description (LLM が読む)**:
> List business outsourcing orders with optional filters. Use this when the accountant asks about specific orders by month, worker, status, or amount range. Do NOT use this for monthly aggregated totals — use `getMonthlyTotals` instead.

**Parameters**:
```ts
{
  monthYearMonth?: string;        // "2026-04"
  workerGithubLogin?: string;
  status?: OrderStatus;
  minAmountJpyc?: number;
  maxAmountJpyc?: number;
  limit?: number;                 // default 50, max 200
}
```

**Returns**: `Order[]` (構造化 JSON、bookkeepingArtifacts 含む)

### 3.2 `getJournalEntries`

**Description**:
> Retrieve journal entries (仕訳) generated by the Bookkeeping Agent for a given month. Use this when the accountant needs to verify or export entries to their accounting software.

**Parameters**:
```ts
{
  yearMonth: string;              // "2026-04"
  workerGithubLogin?: string;
}
```

**Returns**: `JournalEntry[]` (debit / credit / amount / description / dateLocal / orderId reference)

### 3.3 `getWithholdingReport`

**Description**:
> Generate a withholding tax report for a specific worker over a year. The Bookkeeping Agent has already classified each payment; this tool aggregates the existing classifications. For ambiguous classifications it returns "see tax accountant" markers.

**Parameters**:
```ts
{
  workerGithubLogin: string;
  year: number;                   // e.g. 2026
}
```

**Returns**:
```ts
{
  worker: { githubLogin: string; displayName: string; wallet: string };
  year: number;
  totalPaymentsJpyc: number;
  withholdingApplied: { count: number; totalJpyc: number };
  withholdingNotApplied: { count: number; totalJpyc: number };
  needsTaxAccountantReview: { count: number; orderIds: string[] };
  rationale: string;              // Bookkeeping Agent の判定根拠サマリ
}
```

### 3.4 `exportPaymentStatement`

**Description**:
> Get the payment statement (支払調書テンプレ) Markdown for a specific order. The Bookkeeping Agent generates this template; this tool just retrieves it. The template still requires human review before submission to tax authorities.

**Parameters**:
```ts
{
  orderId: string;
}
```

**Returns**:
```ts
{
  orderId: string;
  paymentStatementMarkdown: string;
  generatedAt: string;
  needsHumanReview: boolean;
}
```

### 3.5 `getMonthlyTotals`

**Description**:
> Aggregate total business outsourcing payments for a given month. Use this for high-level accounting review questions like "how much did we spend on outsourcing in April?". For per-order detail use `queryOrders`.

**Parameters**:
```ts
{
  yearMonth: string;              // "2026-04"
}
```

**Returns**:
```ts
{
  yearMonth: string;
  totalOrders: number;
  totalAmountJpyc: number;
  byWorker: { worker: string; orders: number; amountJpyc: number }[];
  byStatus: { status: OrderStatus; count: number }[];
}
```

## 4. Resources

LLM が URI を辿って読める資源。tools/list と並んで MCP の機能。

| URI Pattern | 内容 |
|---|---|
| `gigflow://orders/{orderId}` | 単一 Order の完全 JSON (artifact 含む) |
| `gigflow://accounts/{accountId}` | Account 情報 (worker / company メタデータ) |
| `gigflow://journal/{yearMonth}` | 月次仕訳一覧 (Markdown 整形済み) |
| `gigflow://reports/withholding/{worker}/{year}` | 年間源泉徴収レポート |

### Resource list の応答例

```json
{
  "resources": [
    {
      "uri": "gigflow://journal/2026-04",
      "name": "April 2026 Journal Entries",
      "mimeType": "text/markdown"
    }
  ]
}
```

MCP の prompts 機能で「経理担当者向けテンプレ問合せ」を提供:

### 5.1 `monthly-closing`

**Description**: 月次締め用のチェックリスト問合せ。

**Arguments**:
- `yearMonth` (string, required)

**Generated user message**:

### 5.2 `worker-annual-report`

**Description**: 年間支払調書の準備。

**Arguments**:
- `workerGithubLogin` (string, required)

## 6. 認証実装

### 6.1 JWT 検証

```ts
// packages/mcp-server/src/auth/entra.ts
import { jwtVerify, createRemoteJWKSet } from 'jose';

const JWKS = createRemoteJWKSet(
  new URL('https://login.microsoftonline.com/common/discovery/v2.0/keys'),
);

export async function verifyEntraToken(token: string) {
  const { payload } = await jwtVerify(token, JWKS, {
    audience: process.env.MCP_APP_AUDIENCE,
    issuer: undefined,            // multi-tenant: validate tid manually
  });

  if (!payload.tid) throw new Error('missing tenant id');
  if (!payload.oid) throw new Error('missing object id');

  const roles = (payload.roles ?? []) as string[];

  return {
    tenantId: payload.tid as string,
    userId: payload.oid as string,
    roles,
  };
}
```

### 6.2 Tool 呼び出しガード

```ts
// packages/mcp-server/src/server.ts (の一部)
server.setRequestHandler(CallToolRequestSchema, async (request, extra) => {
  const ctx = await verifyEntraToken(extractBearerToken(extra.request));

  if (!ctx.roles.includes('Accountant') && !ctx.roles.includes('Executive')) {
    throw new McpError(ErrorCode.Forbidden, 'role required: Accountant or Executive');
  }

  const tool = tools[request.params.name];
  if (!tool) throw new McpError(ErrorCode.MethodNotFound, request.params.name);

  return tool.execute(request.params.arguments, ctx);
});
```

すべての tool 関数は `ctx: { tenantId, userId, roles }` を受け取り、Cosmos クエリで必ず `companyId = ctx.tenantId` を使う。

## 7. 接続手順

### 7.1 Claude Desktop

`~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "gigflow": {
      "url": "https://mcp-gigflow-{suffix}.{region}.azurecontainerapps.io/mcp",
      "transport": "streamable-http",
      "auth": {
        "type": "oauth",
        "authorization_url": "https://login.microsoftonline.com/{tenantId}/oauth2/v2.0/authorize",
        "token_url": "https://login.microsoftonline.com/{tenantId}/oauth2/v2.0/token",
        "client_id": "{client_id}",
        "scope": "api://gigflow-mcp/mcp.read"
      }
    }
  }
}
```

### 7.2 VS Code (公式 MCP サポート / Continue / Cline / Cursor)

VS Code は本体に MCP support が入っている (1.86 以降)。`.vscode/mcp.json` に設定。Continue / Cline / Cursor の各拡張の MCP 設定 UI でも同じ URL + Bearer トークンを指定可。

Bearer トークンの取得 (開発時):
```bash
az account get-access-token --resource api://gigflow-mcp --query accessToken -o tsv
```
本番運用は OAuth フロー (Claude Desktop と同等の設定) を利用する。

### 7.3 Copilot Studio から

Copilot Studio の **MCP connector** を有効化し、上記と同じ URL を登録。Topic (`MonthlyReport` 等) から呼ぶ。

## 9. デプロイ

### 9.1 Dockerfile

```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY packages/shared ./packages/shared
COPY packages/mcp-server ./packages/mcp-server
RUN corepack enable && pnpm install --frozen-lockfile
RUN pnpm --filter mcp-server build

FROM node:20-alpine AS runner
WORKDIR /app
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/packages/shared ./packages/shared
COPY --from=builder /app/packages/mcp-server/dist ./dist
COPY --from=builder /app/packages/mcp-server/package.json ./
EXPOSE 3333
CMD ["node", "dist/server.js"]
```

### 9.2 Container Apps へデプロイ

```bash
# packages/mcp-server/scripts/deploy.sh
az containerapp create \
  --name ca-gigflow-mcp \
  --resource-group $RG \
  --environment $CAE_NAME \
  --image ghcr.io/$GITHUB_USER/gigflow-mcp:latest \
  --target-port 3333 \
  --ingress external \
  --registry-server ghcr.io \
  --registry-username $GITHUB_USER \
  --registry-password $GHCR_TOKEN \
  --system-assigned \
  --env-vars \
    COSMOS_ENDPOINT="$COSMOS_ENDPOINT" \
    COSMOS_DATABASE="gigflow" \
    MCP_APP_AUDIENCE="api://gigflow-mcp"

# Cosmos への RBAC
MCP_PRINCIPAL_ID=$(az containerapp identity show --name ca-gigflow-mcp --resource-group $RG --query principalId -o tsv)
az cosmosdb sql role assignment create \
  --account-name $COSMOS_NAME \
  --resource-group $RG \
  --scope $(az cosmosdb show --name $COSMOS_NAME --resource-group $RG --query id -o tsv) \
  --principal-id $MCP_PRINCIPAL_ID \
  --role-definition-id 00000000-0000-0000-0000-000000000001  # Cosmos DB Built-in Data Reader (read-only)
```

> **Reader ロールにする** — 書込権限を渡さない原則。

## 11. Tool description ライティングのガイドライン

LLM が tool を選ぶ精度は description で決まる。以下を必ず満たす:

1. **Use this when ...** で**使うべき場面**を冒頭に
2. **Do NOT use this for ...** で**使ってはいけない場面**を明示
3. パラメータの説明に**自然言語の例**を入れる
4. Returns の構造を文章で要約 (LLM が結果をパースしやすくなる)
5. **副作用を明記** (read-only / writes に分けて)

例 (`queryOrders`):

```ts
{
  description: `
    List business outsourcing orders with optional filters.
    Use this when the accountant asks about specific orders by month, worker, status,
    or amount range (e.g., "Show me April orders for Sato", "Which orders are settled?").
    Do NOT use this for monthly aggregated totals — use 'getMonthlyTotals' instead.
    Read-only.
    Returns up to 200 orders matching the filters, sorted by createdAt DESC.
  `,
  inputSchema: { /* ... */ },
}
```

## 12. ハマりどころ

| 症状 | 原因 | 対処 |
|---|---|---|
| Container Apps で SSE が切れる | プロキシのタイムアウト | Streamable HTTP transport に統一 |
| Claude Desktop が tool を見つけない | `tools/list` が空 | Bearer トークンの roles claim を確認 |
| 別テナントのデータが見えてしまう | tenant scoping 漏れ | `lib/cosmos.ts` で query 段階に強制制約 |
| LLM が間違った tool を呼ぶ | description が曖昧 | "Do NOT use for ..." を必ず入れる |
| Inspector で 401 | audience mismatch | App Registration の `Application ID URI` を確認 |
