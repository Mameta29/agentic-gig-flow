## 3. 発注502の真因（最終確定）

切り分けの結論（ログで実証）:
- 無効/Bearerなしトークン → 401（正常）
- **PMロール無しのトークン → 500 `{"error":"internal"}`**（`requireRole`が普通のErrorをthrow→authErrorResponseがAuthError以外で`internal`返す）
  → **`mameta@MAMETAZK` に Functions の App Role が1つも割り当たっていなかった**のが壁だった。
  → Graph appRoleAssignments で **PM/Accountant/Executive の3ロールを `mameta@MAMETAZK`(userId
    `b322d20f-1110-477b-9698-5a62a3cc8bd6`) に割当済み**。要・再サインイン（トークンにroles反映）。
- PMロール有り → Contract Agent(gpt-4o)が動く → **「JPYCで500」を「報酬1JPYC未満」と誤判定してreject**
  （`contract_failed: 報酬額が 1 JPYC 未満です`）。
  - ローカル実証: 「JPYCで500」→FAIL、「報酬は5000円。期日は2週間後」→**OK(amount 5000, Issue #6作成)**。
  - → **contract.ts のプロンプト改善で対処**（未デプロイ）。または発注文を「報酬は3万円」等の明確表記にすれば通る。

### ⚠️ 私(Claude)のテスト手段のミスリードに注意
セッション中盤、`az account get-access-token --resource api://...` で create を叩いて `HTTP 000`
を得て「ワーカークラッシュ」と誤誘導した。**このazトークンは形式が不適切**（jose検証に合わない）で、
本物の経路（Dashboard の NextAuth トークン）ではない。**502 の検証は必ず Dashboard 経由で**行うこと。
azトークン直叩きは使わない。

## 5. Functions デプロイ手順（確立済み・これを使う）

```bash
cd ~/dev/hackson/agentic-gig-flow
pnpm --filter @gigflow/functions build

rm -rf $PKG && mkdir -p $PKG
cp -R packages/functions/dist $PKG/dist
cp packages/functions/host.json $PKG/host.json
cp packages/functions/package.json $PKG/package.json
cd $PKG
npm pkg delete dependencies.@gigflow/shared
npm install --omit=dev --no-package-lock --ignore-scripts
mkdir -p node_modules/@gigflow/shared
cp -R ~/dev/hackson/agentic-gig-flow/packages/shared/dist node_modules/@gigflow/shared/dist
cp ~/dev/hackson/agentic-gig-flow/packages/shared/package.json node_modules/@gigflow/shared/package.json
func azure functionapp publish func-gigflow-28fa80 --no-build --javascript
```
